# Changelog

All notable changes to yelly-master are documented in this file. The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.2.0] — 2026-04-13

### Added

- **`/yelly-lead` mega-skill (Phase 0 MVP)** — the fifth mega-skill, providing tech-lead capabilities that complement the existing engineering, ops, quality, and team mega-skills.
- **4 new modules:**
  - `adr` — Architecture Decision Records in Nygard format with supersede flow
  - `debt` — Technical debt scanning with hybrid issue-tracker export (gh / glab / jira / linear) or local register fallback
  - `estimate` — PERT-based 3-point estimates with historical calibration via `.yelly/history/estimates.jsonl`
  - `risk` — Severity register (impact × probability) with deploy gate integration
- **Project-level context file `YELLY.md`** — per-project, git-tracked, auto-generated by `/yelly-lead`. Captures Project Snapshot, Active Work, Architecture Decisions, Top Risks, Tech Debt, Decision Log, Open Questions, and Pinned Notes. Section markers (`<!-- yelly-lead: <section> -->`) make updates idempotent.
- **`docs/yelly/` artifacts directory** — durable, append-friendly storage for ADRs (`adr/NNNN-*.md`), estimates (`estimates/<date>-*.md`), risks (`risks/active.md` + `risks/archive/`), and decision archive.
- **`lib/yelly-lead/` library** — reusable functions for frontmatter parsing, tracker detection, YELLY.md section updates, bootstrap context gathering, validation, stats, and sync.
- **3 CLI tools:**
  - `npm run yelly-lead-sync` — regenerate YELLY.md sections from `docs/yelly/` artifacts
  - `npm run yelly-lead-validate` — check YELLY.md schema, ADR filenames, line cap
  - `npm run yelly-lead-stats` — show ADR count, active risks, estimate calibration bias
- **4 templates** — `templates/{YELLY.md,adr.md,estimate.md,risk.md}.tmpl` with placeholder substitution.
- **`docs/yelly-lead/OVERVIEW.md`** — user guide for the new mega-skill.
- **`docs/yelly-lead/YELLY_MD_SCHEMA.md`** — schema reference for YELLY.md frontmatter and sections.
- **`migrations/v0.2.0-yelly-lead.ts`** — no-op migration shim for upgraders from 0.1.0.

### Changed

- `README.md` — architecture diagram now shows 5 mega-skills + 23 modules; modules table includes the 4 new yelly-lead modules; roadmap updated.
- `setup` — registers `/yelly-lead` skill in addition to the existing 4 mega-skills.
- `config/defaults.yaml` — adds default blocks for `adr`, `debt`, `estimate`, `risk`.

### Tests

- Added unit, static, and integration tests across yelly-lead lib and modules. Total suite: **183 tests** (was 61 in 0.1.0).

## [0.1.0] — 2026-03-04

Initial release: 4 mega-skills (yelly-code, yelly-ops, yelly-quality, yelly-team), 19 modules, multi-host setup script, GitHub Actions CI.
