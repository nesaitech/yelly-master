# Changelog

All notable changes to 8hour-master are documented in this file. The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.3.0] — 2026-04-13

### Changed

- **BREAKING — project rebranded from `yelly` to `8hour`.** Every user-facing surface, slash command, filename, library module, config key, CLI binary, template, and docs reference is renamed:
  - Slash commands: `/yelly-code`, `/yelly-ops`, `/yelly-quality`, `/yelly-team`, `/yelly-lead` → `/8hour-code`, `/8hour-ops`, `/8hour-quality`, `/8hour-team`, `/8hour-lead`
  - Project context file: `YELLY.md` → `8HOUR.md`
  - Artifacts directory: `docs/yelly/` → `docs/8hour/`
  - Internal state directory: `.yelly/` → `.8hour/`
  - npm package: `yelly-master` → `8hour-master`
  - CLI binaries: `yelly-config`, `yelly-update-check`, `yelly-repo-mode`, `yelly-lead-sync`, `yelly-lead-validate`, `yelly-lead-stats` → `8hour-*`
  - Library paths: `lib/yelly-lead/*` → `lib/8hour-lead/*`
  - Config file: `config/yelly-lead.yaml` → `config/8hour-lead.yaml` (and `yelly_lead:` / `yelly_md:` keys → `8hour_lead:` / `8hour_md:`)
  - Frontmatter schema keys: `yelly_version` / `yelly_lead_version` → `8hour_version` / `8hour_lead_version`
  - User config dir: `~/.yelly-master/` → `~/.8hour-master/`
- **Upgrade path** for existing users: rename your project-local `YELLY.md` → `8HOUR.md`, move `docs/yelly/` → `docs/8hour/`, move `.yelly/` → `.8hour/`, move `~/.yelly-master/` → `~/.8hour-master/`, and re-run `./setup`. No automated migration — the rename is cosmetic and internal file formats are unchanged.

## [0.2.0] — 2026-04-13

### Added

- **`/8hour-lead` mega-skill (Phase 0 MVP)** — the fifth mega-skill, providing tech-lead capabilities that complement the existing engineering, ops, quality, and team mega-skills.
- **4 new modules:**
  - `adr` — Architecture Decision Records in Nygard format with supersede flow
  - `debt` — Technical debt scanning with hybrid issue-tracker export (gh / glab / jira / linear) or local register fallback
  - `estimate` — PERT-based 3-point estimates with historical calibration via `.8hour/history/estimates.jsonl`
  - `risk` — Severity register (impact × probability) with deploy gate integration
- **Project-level context file `8HOUR.md`** — per-project, git-tracked, auto-generated by `/8hour-lead`. Captures Project Snapshot, Active Work, Architecture Decisions, Top Risks, Tech Debt, Decision Log, Open Questions, and Pinned Notes. Section markers (`<!-- 8hour-lead: <section> -->`) make updates idempotent.
- **`docs/8hour/` artifacts directory** — durable, append-friendly storage for ADRs (`adr/NNNN-*.md`), estimates (`estimates/<date>-*.md`), risks (`risks/active.md` + `risks/archive/`), and decision archive.
- **`lib/8hour-lead/` library** — reusable functions for frontmatter parsing, tracker detection, 8HOUR.md section updates, bootstrap context gathering, validation, stats, and sync.
- **3 CLI tools:**
  - `npm run 8hour-lead-sync` — regenerate 8HOUR.md sections from `docs/8hour/` artifacts
  - `npm run 8hour-lead-validate` — check 8HOUR.md schema, ADR filenames, line cap
  - `npm run 8hour-lead-stats` — show ADR count, active risks, estimate calibration bias
- **4 templates** — `templates/{8HOUR.md,adr.md,estimate.md,risk.md}.tmpl` with placeholder substitution.
- **`docs/8hour-lead/OVERVIEW.md`** — user guide for the new mega-skill.
- **`docs/8hour-lead/8HOUR_MD_SCHEMA.md`** — schema reference for 8HOUR.md frontmatter and sections.
- **`migrations/v0.2.0-8hour-lead.ts`** — no-op migration shim for upgraders from 0.1.0.

### Changed

- `README.md` — architecture diagram now shows 5 mega-skills + 23 modules; modules table includes the 4 new 8hour-lead modules; roadmap updated.
- `setup` — registers `/8hour-lead` skill in addition to the existing 4 mega-skills.
- `config/defaults.yaml` — adds default blocks for `adr`, `debt`, `estimate`, `risk`.

### Tests

- Added unit, static, and integration tests across 8hour-lead lib and modules. Total suite: **183 tests** (was 61 in 0.1.0).
